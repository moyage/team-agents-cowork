# 操作指南 (Guides)

本章节是 Team Agents Cowork 的实战指导手册。在掌握了基本安装与核心概念之后，这里的指南将深入指导您如何将框架应用于实际的工程和业务场景中。

无论您是要从头开始设计一个新的编排流水线，还是想要配置一个多职能混合的智能体研发团队，本章节都提供了结构化的步骤与最佳实践。

## 目录与导读

本章节包含以下进阶操作指南：

- **[编写工作流 (authoring-workflows.md)](./authoring-workflows.md)**
  这是定制系统行为的核心指南。详细讲解如何从零开始设计自定义工作流，包括 DAG（有向无环图）节点定义、依赖关系配置、如何利用 `blackbox` 与 `orchestrated` 模式，以及编写符合验证 Schema 的 YAML 文件的最佳实践。

- **[团队协作实战 (team-collaboration.md)](./team-collaboration.md)**
  探讨如何构建基于 L2 (智能体节点) / L3 (编排大脑) 双层拓扑架构的多智能体协作系统。内容涵盖连接异构智能体、规范跨职能角色（如产品/开发/测试智能体）之间的通讯边界，以及状态同步策略。

## 建议阅读路径
如果您希望快速让系统适应您的特定业务需求，建议优先阅读**编写工作流**指南。而如果您的目标是打造一个具有复杂角色交互的大型智能体集群，则**团队协作实战**将为您提供必需的架构思维指导。
