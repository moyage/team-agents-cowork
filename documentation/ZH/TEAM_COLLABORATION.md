# 团队协作说明

在团队模式下，该协议作为一个极其严格的异步同步机制发挥作用。

## 能力建模 (Capability Modeling)
专属的 Orchestrator Agent (编排者) 会读取需求，将其拆解为多个受限的 JSON 合约。
- 编排者会读取环境的能力注册表。
- 任务 A (前端UI) 被分配给 Trae/Cursor 用户执行。
- 任务 B (耗时测试) 被分配给 OpenCode 后台守护进程执行。

## 冲突预防
因为每一个任务都在 `Pre-Gate` 阶段强制规定了且被审核了 `allowed_files` 物理边界，并行执行在数学上被保证绝不会发生相同文件的写入冲突。
