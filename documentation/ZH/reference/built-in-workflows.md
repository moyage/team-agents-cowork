# 17 个内置工作流

本框架内置了 17 个高频 DAG 工作流模板。

| ID | 描述 |
|---|---|
| `assist` | 常规问答、Debug 与代码探索 |
| `fix-github-issue` | 全链路: 计划 -> 修复 -> 测试 -> PR -> 审查 |
| `idea-to-pr` | 灵感 -> 计划 -> 实现 -> PR -> 并行审查 |
| `plan-to-pr` | 执行一个已有的设计文档 -> 提交 PR |
| `issue-review-full` | 针对 GitHub Issue 的深度联合审查 |
| `smart-pr-review` | 基于复杂度的智能目标审查 |
| `comprehensive-pr-review` | 5 个并行审查者 + 自动修复 |
| `create-issue` | 调查代码库并正式起草 Issue |
| `validate-pr` | 深度分支验证与测试 |
| `resolve-conflicts` | 智能解决 Git 合并冲突 |
| `feature-development` | 直接根据设计实现特性 |
| `architect` | 架构扫描与技术债降低 |
| `refactor-safely` | 带类型检查的非破坏性重构 |
| `ralph-dag` | 迭代式 PRD 故事点实现 |
| `remotion-generate` | 生成 Remotion 视频组件 |
| `test-loop-dag` | 迭代式故障分析与修复套件 |
| `piv-loop` | 包含人工检查点的计划-实现-验证循环 |
