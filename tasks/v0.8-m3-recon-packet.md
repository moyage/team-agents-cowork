# V0.8-M3 Reconnaissance: Built-in Workflow Porting (The "Archon 17")

## 1. Objective
Translate the power of Archon's dynamic workflow routing into the `team-agents-cowork` L2/L3 federated architecture. We will port Archon's 17 default workflows into our custom YAML DAG format. 

## 2. Architectural Paradigm Shift
Unlike Archon, where nodes are direct LLM calls executed by a central orchestrator, our YAML nodes act as **Intent Generators**. 
When the LLM Router selects a workflow (e.g., `idea-to-pr.yaml`), the engine reads the DAG nodes. For a node of type `ai_execution`, the engine does NOT call the LLM; instead, it generates an `execution-contract.json` payload and places it in the target `.agent-state/tasks/` directory, waiting for a registered local agent (Cursor, Opencode) to *pull* it, execute it, and submit it for Gatekeeper review.

## 3. Target Scope (The "Archon 17" Port)
We will create `templates/workflows/*.yaml` containing the following 17 default templates (which `agent-protocol-init` will later copy to target repositories):
1. `assist` (General Q&A, debugging)
2. `fix-github-issue` (Issue -> PR with smart review)
3. `idea-to-pr` (Feature idea -> PR with multi-review)
4. `plan-to-pr` (Execute existing plan -> PR)
5. `issue-review-full` (Multi-agent issue review)
6. `smart-pr-review` (Targeted PR review agents)
7. `comprehensive-pr-review` (5 parallel reviewers)
8. `create-issue` (Investigate and create issue)
9. `validate-pr` (Thorough testing of main vs feature)
10. `resolve-conflicts` (Detect, resolve, validate)
11. `feature-development` (Implement from plan)
12. `architect` (Architectural sweep, codebase health)
13. `refactor-safely` (Safe refactoring with type-checks)
14. `ralph-dag` (PRD implementation iterative loop)
15. `remotion-generate` (Video composition modification)
16. `test-loop-dag` (Iterative test fixing loop)
17. `piv-loop` (Plan-Implement-Validate guided loop)

## 4. Constraints
- **In-Scope:** Defining the 17 YAML templates in `templates/workflows/`.
- **Out-of-Scope:** Implementing the actual Node.js DAG execution runner (that parses dependencies and generates contracts at runtime). This is an authoring phase, not an engine building phase. Modifying active workflow state semantics outside of M3 progression is forbidden.
