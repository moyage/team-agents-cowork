# V0.8.7 Backlog

This backlog synthesizes findings from the V0.8.6 Code Review Report and the Solo/Team Workspace Mode Switch Design Document.

## 1. Features & Enhancements

**Solo/Team Workspace Mode Switch**
*   **Configuration Integration:** Implement workspace mode configuration (`workspace.mode: "solo" | "team"`) in `manifest.yaml` or `profiles/*.yaml`. Define concurrency limits for Solo mode and isolation parameters/endpoints for Team mode.
*   **CLI Parameter Override:** Add a `--mode=solo|team` flag to the `team-agents-cowork run` command for dynamic overriding.
*   **Orchestrator Routing Logic:**
    *   **Solo Mode:** Spawns subagent processes locally, shares the workspace directory, and manages concurrent file edits via Git/FS locks.
    *   **Team Mode:** Delegates tasks via API or remote container invocation. Syncs workspace state pre- and post-execution based on `delegation_mode`.
*   **Schema Updates:** Update `schemas/execution-contract.schema.json` to include the new `workspace_mode` parameter.
*   **CLI Runtime Check:** Implement the mode check and corresponding branch logic in the core CLI runtime.

## 2. Architecture & Protocol

*   **Dynamic Participant Registry:**
    *   Remove hardcoded agent names (`hermes-local`, `opencode-sisyphus`) from the default `registry.json` scaffold in `bin/agent-protocol-init`. Replace with a minimal, capability-agnostic template.
    *   Remove hardcoded agent implementations/brands (`opencode-core-only`, `cursor-human-in-loop`, `architect-core`) from workflow templates (`architect.yaml`, `custom-deploy-pipeline.yaml`, `idea-to-pr.yaml`). Use abstract capability requirements instead.
*   **L3 Gatekeeper (Review Daemon):**
    *   Upgrade `bin/agent-protocol-review` from a mock/simulated SubAgent (which hardcodes `reviewerAgentId = "mock-isolated-reviewer"`) to a functional SubAgent bridge, or explicitly mark it as `TODO/Mock` in the CLI output.

## 3. Security

*   **Shell Injection Remediation:** Refactor `bin/agent-protocol-review` to use `fs.writeFileSync` directly instead of passing `artifactFullPath` through `execSync` with inline Node evaluation.
*   **Path Traversal Prevention:** Validate and sanitize `expectedArtifact` in `bin/agent-protocol-review` before calling `fs.mkdirSync` to prevent directory traversal outside the `PROJECT_ROOT`.
*   **Contract File Boundary Check:** Add a `path.startsWith(resolvedBase)` boundary check for `contract_path` in `verify_execution_compliance` (`src/mcp-server/src/index.ts`) to match existing robust checks.
