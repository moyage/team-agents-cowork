# M3 Reconnaissance Note
## Server Capability Baseline
- **Current Tools**: `get_iteration_state` (reads `workflow/iteration-state.json`), `verify_execution_compliance` (compares git diff --name-only against a contract's allowed_files).
- **Boundary Status**: 100% Read-only operations. No write capabilities exist in the MCP server.
- **Coverage Check**: Protocol validation currently covers architectural drift detection via file paths. It lacks semantic validation (content diffs) and workflow state introspection (e.g., dispatch routing).

## M3 Candidate Directions
1. **Semantic Diff Context Tool**: Expand the MCP server to provide actual code diff content for Reviewer Agents. Currently, Reviewers must run raw shell commands to get `git diff`. Providing `get_execution_diff` centralizes this into the MCP protocol safely.
2. **Workflow Routing Inspection**: A read-only tool to safely parse and return the canonical `workflow/dispatch.json`, allowing planner/orchestrator agents to securely understand the active gate and constraints without grep/cat.
3. **Artifact Discovery Tool**: A read-only tool that queries `tasks/*.json` to retrieve the latest execution results and review decisions programmatically.

## Recommended Focus
**Combined Context API (Semantic Diff + Dispatch Inspector)**
- *Why:* It directly empowers the orchestrator and reviewer by standardizing their context gathering through the MCP protocol, shifting them away from brittle, shell-dependent data fetching.
- *Value:* Hardens the Layer-2 / Layer-3 interface. Sub-agents (like GPT-5.4 Reviewer) can call an MCP tool to get the contract and the diff simultaneously instead of running bash scripts.
- *Bounded Scope:* Strictly read-only file reading and `git diff` operations. 
- *Overlap/Drift Avoidance:* Does not introduce mutators, write APIs, or redesign the identity of the Reviewer.

## Governance Guardrails
- Must NOT implement state transition mutators (`update_dispatch`). State updates remain a local execution artifact process.
- Must NOT implement "auto-approval" decision trees within the MCP server.
- Must keep artifact file paths configurable to respect `allowed_artifacts` rules introduced in M2 Retrospective.
