# V0.7 Kick-off Packet: The Protocol-First Collaboration System

## 1. Background & Context
Iteration V0.6 successfully established repository hygiene, unified documentation, and hardened the foundational Read-Only Context APIs. It closed with a robust, mathematically verified Gatekeeper protocol.

**V0.7 is a brand-new iteration.** It elevates the framework from a "repository protocol tool" to a comprehensive, multi-agent, multi-AI coding collaboration system. It shifts the focus from structural cleanup to behavioral protocols, environment adaptation, and governed self-evolution.

## 2. North Star Objective
> "Construct a protocol-first collaboration system that serves personal and team domains, supports multi-agent/multi-AI coding environments, operates with low invasiveness and low cognitive load, and possesses capabilities for auditable, adaptable, and governed self-evolution."

## 3. Core Requirements & Carry-overs
### V0.6 Carry-overs
1. **State Separation:** Phased migration of `.agent-state/` to decouple the framework source from user execution state.
2. **Legacy Tooling Deprecation:** Phased deprecation map for obsolete Python validation scripts.
3. **SubAgent Review Automation:** Reducing friction by automating the invocation of Gatekeeper sub-agents on state transitions.

### New Core Requirements
1. **Multi-Agent Collaboration Framework:** Protocols for Personal Mode, Team Mode, task decomposition, and role boundaries (Executor, Reviewer, Coordinator).
2. **Governed Self-Evolution Loop:** Systemic capabilities for retrospective analysis, failure taxonomy, and proposing improvements, bounded by strict Phase Gating rules (no unreviewed self-modifications).
3. **Environment Discovery & Adaptation:** Capability modeling, fingerprinting, and adapter evolution for diverse AI Coding ecosystems (Opencode, OMOC, Superpowers, Openspec).

## 4. Architectural Principles
1. **Low Cognitive Load:** Simplified workflows; complexity is handled by the protocol, not the user.
2. **Low Invasiveness:** Do not force a single IDE or agent environment. Adapt to existing tools.
3. **Protocol-First:** The collaboration contract and gating rules are defined before implementation.
4. **Adapter-Based:** Environment-specific capabilities are modular and optional.
5. **Minimal Mandatory Core:** The blocking gates are minimal; extensions are layered.
6. **Governed Autonomy:** Agents can propose and prepare changes autonomously, but physical implementation requires independent Gatekeeper/Human approval.

## 5. Execution Discipline
This iteration operates under **Milestone-Level Autonomy**. The framework will define, execute, and review bounded execution contracts per milestone, escalating to the user only on scope drift, state model mutations, mass deletions, or architectural forks.
