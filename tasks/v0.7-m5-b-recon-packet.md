# V0.7-M5-B Reconnaissance: Legacy Tooling Deprecation Roadmap

## 1. Objective
Design the deprecation roadmap for the legacy Python tooling (`scripts/*.py`). Define the criteria for deprecating, retaining, or removing these scripts as the framework fully transitions to the Node.js MCP server model.

## 2. Analytical Scope
- **Target Files:** `scripts/doctor.py`, `scripts/verify_protocol.py`, `scripts/sync_to_target.py`, `scripts/validate_workflow_state.py`, `scripts/build_demand_pool.py`, `scripts/verify_integrity.py`, `scripts/update_integrity_manifest.py`.
- **Deprecate vs. Retain vs. Remove Criteria:**
  - *Remove:* Scripts whose logic is now 100% covered by the `src/mcp-server/`.
  - *Retain:* Scripts that perform low-level bootstrap tasks (e.g., `install.sh` helpers) that cannot yet be fully handled by the NPM CLI package.
  - *Deprecate:* Scripts that are redundant but might still be referenced by legacy external CI pipelines until V0.8.
- **Workflow Semantic Integrity:** Removing tools must not break existing `dispatch.json` state validation if someone tries to use a legacy CLI command.

## 3. Constraints
- **In-Scope:** Drafting the roadmap design document (`tasks/v0.7-m5-b-legacy-tooling-roadmap.md`).
- **Out-of-Scope:** Deleting any actual Python files. Modifying active `dispatch.json` state semantics or MCP logic.
