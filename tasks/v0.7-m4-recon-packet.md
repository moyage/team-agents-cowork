# V0.7-M4 Reconnaissance: Environment Discovery & Capability Modeling

## 1. Objective
Design the abstractions and registries required for the framework to dynamically discover, model, and adapt to heterogeneous AI Coding environments (e.g., Opencode, OMOC, Superpowers) without forcing a single IDE or CLI.

## 2. Analytical Scope
- **Environment Fingerprint Model:** How does the system passively detect what tool the user or agent is running? (e.g., specific env vars, presence of config files like `.cursorrules`).
- **Capability Registry:** How do we define what an environment *can* do? (e.g., `supports_mcp`, `supports_background_processes`, `can_read_files`).
- **Compatibility / Drift Analysis Method:** How do we know when an adapter (like the Cursor prompt) is outdated or failing to trigger the MCP tools?
- **Adapter Abstraction Rules:** How to decouple the core protocol's requirements from the environment's specific way of taking system prompts.

## 3. Constraints
- **In-Scope:** Producing a design document (`tasks/v0.7-m4-environment-adaptation.md`).
- **Out-of-Scope:** Writing actual adapter detection code in the MCP Server or CLI tools. Modifying `workflow/` state semantics. Restricting the framework to only support one tool.
