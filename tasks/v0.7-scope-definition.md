# V0.7 Scope Definition

The scope of Iteration V0.7 is defined strictly by the goal of elevating the Team Agents Cowork framework into a multi-agent, multi-AI coding collaboration protocol system. The scope encompasses the following defined boundaries and exclusions.

## 1. In Scope
- **Collaboration Model:** Define protocols for Personal Mode, Team Mode, multi-agent task decomposition, and role boundaries (Executor, Reviewer, Gatekeeper, Coordinator). Define collaboration contracts, acceptance standards, and auditability.
- **Protocol & Workflow:** Establish unified models for harness, workflow, orchestration, execution, evaluation, and acceptance. Define task lifecycles, artifact semantics, review gating semantics, and state transition models.
- **Self-Evolution Loop:** Implement feedback ingestion design, failure taxonomy, retrospective loops, improvement proposal pipelines, and governed self-repair models.
- **Environment Discovery & Adaptation:** Model environment fingerprints, create a capability registry, define compatibility/drift analysis methods, and abstract adapter rules for tools like Opencode, OMOC, Superpowers, and Openspec.
- **Carry-over Architecture Tracks:** Establish roadmaps for `.agent-state/` state separation, legacy tooling deprecation, and SubAgent review automation.

## 2. Out of Scope (Strict Exclusions)
- **Uncontrolled Mass Migration:** Directly executing large-scale physical migrations to `.agent-state/` without prior design and explicit approval.
- **Mass Deletion:** Deleting large quantities of legacy tooling without a mapped deprecation phase and user approval.
- **Rewriting Workflow State Semantics:** Modifying the fundamental meaning of the state model without explicit authorization.
- **Unreviewed "Auto-Update":** Implementing automatic, unconditional self-modification capabilities that bypass the Independent Reviewer / Gatekeeper protocol.
- **Single-Toolchain Lock-in:** Developing deep, exclusive adaptations that bind the framework to only one specific AI Coding environment (e.g., only Opencode), violating the "Low Invasiveness" principle.
- **Uncontrolled Refactoring:** Using V0.7 Kickoff as an excuse to execute large, unbounded refactoring of the MCP server or framework core code.
- **"Low Invasiveness" = "No Protocol":** Relaxing the fundamental L0-L6 architecture rules. The protocol must remain physically verified, but the integration friction must be low.
