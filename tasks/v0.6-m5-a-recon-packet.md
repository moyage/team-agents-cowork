# V0.6-M5-A Reconnaissance: Tooling Deprecation & Read-Only API Hardening

## 1. Objective
Assess the technical debt within the `src/mcp-server/src/index.ts` path resolution and inventory legacy Python scripts for deprecation.

## 2. Reconnaissance: Path Resolution Fragility
- **Current State:** 
  The MCP server currently uses hardcoded relative paths based on `__dirname`:
  - `WORKFLOW_STATE_PATH = path.resolve(__dirname, "../../workflow/iteration-state.json")`
  - `projectRoot = path.resolve(__dirname, "../../")` (used in `get_execution_diff` and `verify_execution_compliance`)
  - `dispatchPath = path.resolve(__dirname, "../../../workflow/dispatch.json")` (used in `get_dispatch_state`)
- **Vulnerability:** 
  If the MCP server is installed or run from anywhere other than the exact framework repository directory, `__dirname` will lead to incorrect paths. When an external project uses this framework, their `workflow/` and `tasks/` directories live in their project root, NOT relative to the global framework installation.
- **Proposed Solution:** 
  The MCP server should resolve paths relative to a configurable `PROJECT_ROOT` environment variable, or default to `process.cwd()`. This completely removes the fragile `__dirname` relative jumps.

## 3. Reconnaissance: Legacy Tooling Deprecation
- **Inventory:** 
  The `scripts/` directory contains: `doctor.py`, `verify_protocol.py`, `sync_to_target.py`, `build_demand_pool.py`, `validate_workflow_state.py`, `verify_integrity.py`, `update_integrity_manifest.py`, etc.
- **Assessment:**
  These scripts were used for V0.3/V0.4 phase gating. V0.5 moved validation to the MCP Server.
- **Action:** 
  Per the user constraints ("默认只做盘点与弃用评估，不做实际删除"), we will NOT delete these scripts. We will list them as technically deprecated by the MCP server but preserve them.
