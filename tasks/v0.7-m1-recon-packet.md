# V0.7-M1 Reconnaissance: Protocol Reframing & Scenario Modeling

## 1. Objective
Define the foundational protocol rules, role models, and scenario boundaries for Personal Mode and Team Mode multi-agent collaboration, creating the blueprint for V0.7.

## 2. Current State Analysis
- **V0.6 State:** The framework successfully enforces a rigid L2/L3 dual-track workflow (Executor vs. Gatekeeper) using a single MCP Server. However, it implicitly assumes a 1:1 human-to-agent mapping.
- **The Gap:** There is no formalized protocol for how multiple agents (e.g., an Orchestrator decomposing tasks, multiple Executors working in parallel, a dedicated Reviewer sub-agent) interact with the state machine. Personal vs. Team modes are conceptually mentioned but lack formalized interaction constraints.

## 3. Scope Definition
- **In-Scope:** 
  - Design the `Core Collaboration Protocol`: Define the minimal mandatory core (state, contract, artifacts) vs. the heterogeneous adapter layer.
  - Define `Roles`: Coordinator, Executor, Reviewer, Gatekeeper.
  - Model `Scenarios`: Personal Mode vs. Team Mode.
- **Out-of-Scope:**
  - Writing code to enforce these roles (deferred to later milestones).
  - Modifying existing `workflow/` or `documentation/` files (this is a design/planning milestone, artifacts will be generated in `tasks/`).
