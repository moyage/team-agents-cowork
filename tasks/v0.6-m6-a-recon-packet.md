# V0.6-M6-A Reconnaissance: State Separation & Synthesis

## 1. Objective
Perform a bounded design, comparative synthesis, and impact analysis regarding the separation of execution state from the framework codebase, building upon insights from `labs-teamwork-spec`.

## 2. Active Reference Scan
- **Findings:** 
  - `README.md` correctly points to `documentation/`.
  - `bin/agent-protocol-init` injects adapters mapping to `documentation/AI_GUIDES/SYSTEM_PROMPT.md`.
  - `workflow/iteration-state.json` tracks `tasks/` artifacts from V0.5 and V0.6.
- **Impact:** 
  - Archiving V0.4 and V0.5 artifacts will require removing them from `iteration-state.json` tracking arrays to avoid broken references.
  - Redirecting `docs/` and `protocol/` via stub files instead of hard deletion ensures that old external hyperlinks or integrations fail gracefully.

## 3. Comparative Synthesis vs. `labs-teamwork-spec`
- **`labs-teamwork-spec`:** Employs an NPM CLI package model where the tool source is cleanly decoupled from the target project `.agent-laws` rules. It relies on the IDE's native context without a physical state machine lock.
- **`team-agents-cowork`:** Implements an active MCP Server enforcing a physical `dispatch.json` Phase Gate. 
- **Synthesis Action:** We must adopt the CLI generation pattern from `labs` (begun in M4 with `agent-protocol-init`) to generate `workflow/` and `tasks/` in the *user's* repo, but retain our MCP Server to physically enforce those gates.

## 4. State/Framework Separation Design
- **Current Problem:** The framework repository serves simultaneously as the engine source code and the execution log (`tasks/`, `workflow/`).
- **Proposed Architecture:**
  - The repository should only contain `bin/`, `src/mcp-server/`, `documentation/`, and `adapters/`.
  - When `bin/agent-protocol-init` is run in a user's target repo, it creates `.agent-state/workflow/` and `.agent-state/tasks/`.
  - The MCP Server `PROJECT_ROOT` environment variable will securely map the server's context to the user's `.agent-state` directory.
- **Execution Plan (Deferred to Future Iteration):** Given the massive blast radius of redefining the active MCP server paths and altering all existing CI hooks, actual physical migration is deferred. M6-B will only execute local archiving.

## 5. M6-B Cleanup & Archive Manifest
- **Targets for Archival:**
  - `tasks/v0.4-*` and `tasks/v0.5-*` artifacts -> Move to `.archive/tasks-archive/`
- **Targets for Stubbing:**
  - Replace contents of `docs/*.md` and `protocol/*.md` with a deprecation notice pointing to `documentation/`.

## 6. Constraints
M6-A is strictly design and analysis. No deletions or migrations will occur until M6-B.
