# V0.5 Kickoff Planning Packet: Native MCP Server Upgrade

## Background / Context
The project has successfully closed V0.4, validating the MVP workflow orchestrator. The objective now transitions to V0.5, which upgrades the Solo Track to a Native TypeScript Model Context Protocol (MCP) Server. This replaces legacy Python CLI wrappers to natively integrate with IDE UIs and enforce schema validation at the parameter level.

## Current Baseline Status
V0.4 baseline restored and structurally verified. Orphaned schemas (`execution-contract.schema.json`, `execution-result.schema.json`, `review-decision.schema.json`, `iteration-state.schema.json`) and the `validate_workflow_state.py` script have been strictly recovered. Unauthorized execution states have been purged. The repository is locked in a clean, strictly compliant pre-execution state.

## V0.5 Objectives
1. Implement a canonical TypeScript-based Native MCP Server for the Team Agent Collaboration Protocol.
2. Formally deprecate and tombstone legacy executable code (M3).
3. **Critical Meta-Objective**: V0.5 is both a functional iteration and a true validation of the `team-agents-cowork` v0.4 orchestration and workflow mechanism under strict enforcement.

## Guiding Principles
- **Terminology Purge**: The terms "MTP" and "MTP V3" are strictly deprecated globally. Use "Team Agent Collaboration Protocol" exclusively.
- **Open Source / Public Repo**: The repository must remain Public on GitHub (`moyage/team-agents-cowork`).
- **Agent/Team Collaboration Philosophy**: Heterogeneous local AI tools coexist, unified via Markdown artifacts and strict JSON Schema. 5-Step Strict Workflow: Clarify -> Spec -> Plan -> Build -> Verify -> Audit & Merge -> Retro. Hard Verification: Verification must be physically independent. Reject "Agentic Sycophancy."
- **Strict Consistency ("Anti-Hype") Policy**: Documentation must exactly match physical implementation. No exaggerating capabilities.
- **File-Driven Coordination**: Move away from chat-based context passing. Interactions must use strictly validated JSON contract files. Physical files MUST exist on disk before state can advance.
- **Execution Boundary & Contract Rules**:
  - No code execution or file modification without a predefined, validated `execution-contract.json`.
  - Agents are restricted to editing only files explicitly listed in the contract's `allowed_files`.
  - **Self-issuing review decisions is strictly forbidden; agents act as `Agent-Executor` only.**
  - Iteration state (`iteration-state.json`) cannot be advanced without explicit human/reviewer authorization.
- **M1 Execution Gate**: **M1 must only start after independent human/reviewer approval of this planning packet.**

## Scope
- Scaffold a standard Node.js/TypeScript MCP Server in `src/mcp-server/`.
- Implement a read-only probe (`get_iteration_state`) using `@modelcontextprotocol/sdk`.
- Execute strictly via V0.4-compliant JSON execution contracts.

## Non-Goals
- Full feature parity with all legacy Python scripts in M1.
- Expanding the scope to multi-agent concurrent write operations (read-only probe first).

## Milestone Plan
### M1: Base MCP Server Scaffold
Focus purely on the base MCP Server scaffold and a single read-only probe (`get_iteration_state`), strictly driven by JSON execution contracts. Bound strictly to `src/mcp-server/`.
### M2: Core Workflow Tools Migration
Port remaining core validation and state transition logic into the native TS MCP tools.
### M3: Tombstone Deprecation
Replace obsolete executable Python scripts with explicit print/error statements pointing to the new architecture rather than silent deletion.

## Deliverables by Milestone
- **M1**: `package.json`, `tsconfig.json`, `src/index.ts` (Native MCP Server scaffold). M1 `execution-contract.json` & `execution-result.json`.
- **M2**: Extended TypeScript implementations for `validate_workflow_schema`, `suggest_next_step`.
- **M3**: Tombstone wrappers for `scripts/mcp_tools.py` and other replaced legacy utilities.

## Dependencies / Assumptions
- The execution environment has Node.js and `npm`/`pnpm` available.
- `@modelcontextprotocol/sdk` is accessible via public npm registries.

## Risks / Constraints
- **Risk**: Agent scope drift during NPM initialization or TS configuration.
- **Constraint**: Must strictly adhere to `allowed_files` in the `execution-contract.json`. The array construction constraint (`[]` instead of `{}`) must be respected in all JSON artifacts.
