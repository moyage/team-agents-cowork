# Team Agents Cowork V0.6 Kickoff Packet

## 1. Background & Context
Iteration V0.5 successfully introduced a mathematically verified, MCP-driven gating protocol. However, it also exposed fragility in path resolution, mounting architectural debt in the repository's directory structure, and a steep learning curve for both human collaborators and AI integration environments.

**V0.6 is a brand-new iteration.** It is not a patch or amendment to V0.5. Its primary mandate is to consolidate these learnings and execute a deep normalization of the project's structure, documentation, and bootstrap mechanisms.

## 2. Iteration Goal
**Objective:** Enhance system reliability, clarify repository structure, dramatically improve documentation consumption, simplify installation/deployment, and increase operability for human teams and AI environments—all while maintaining the strict governance discipline established in V0.5.

## 3. Imported Carry-Overs & Inputs
The following technical debts and requirements are explicitly imported into the V0.6 scope:
1. **Path Resolution Fragility:** `get_iteration_state` and `get_execution_diff` use potentially unsafe relative paths (from V0.5 Retro).
2. **Reviewer Output Delivery:** Optimizing the bridge connecting Sub-Agent context directly to MCP endpoints.
3. **Approval Gating Automation:** Generalizing hooks for `_review_pending` state transitions.
4. **Repository Normalization:** Purging/archiving temporary files, `tasks/` artifacts, and consolidating 15+ root directories.
5. **Docs Center & Samples:** Establishing unified `HUMAN_GUIDES`, `AI_GUIDES`, and `SAMPLES` for clear cognitive onboarding.
6. **Bootstrap / Installability:** Creating 1-click install, deployment, and configuration scripts for individuals and teams.
7. **Comparative Synthesis:** A structural comparison with the locally available `/Users/mlabs/Programs/labs-teamwork-spec` framework.

## 4. Guiding Principles & Constraints
- **Separation of Concerns:** The source framework must be cleanly decoupled from execution state logs (e.g., the `tasks/` directory bloating).
- **Execution Discipline:** All execution phases (M2-M6) must continue to follow the Contract -> Review -> Implementation -> Review -> Closure phase gating model.
- **M5 and M6 are Mandatory:** Repository Normalization (M5) and Installability/Comparative Synthesis (M6) are core pillars of this release, not optional backlog items.
- **No Kickoff Execution:** This phase (M1 Kickoff) is strictly for definition, reconnaissance, and milestone mapping. No files will be moved, and no MCP server code will be changed during M1.
