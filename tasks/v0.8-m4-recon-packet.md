# V0.8-M4 Reconnaissance: SubAgent Review Automation Daemon

## 1. Objective
Eliminate the manual friction of invoking the Gatekeeper in the Personal Domain. In V0.7 and previous milestones, the Human Coordinator had to manually run a terminal command (e.g., `opencode run -m "openai/gpt-5.4"`) to perform Contract Reviews (`contract_review_pending`) and Result Reviews (`result_review_pending`). M4 implements a background daemon/hook (`agent-protocol-review`) to autonomously trigger this isolated review process whenever the state machine transitions to a pending review gate.

## 2. Analytical Scope

### A. The Automation Mechanism (CLI Hook)
- **Target File:** `bin/agent-protocol-review` (Node.js script).
- **Trigger Condition:** Can be run manually, attached to a Git `pre-commit` hook, or polled by a background process.
- **Logic Flow:**
  1. Read `.agent-state/workflow/dispatch.json`.
  2. If `current_gate` is NOT `contract_review_pending` or `result_review_pending`, exit silently (0).
  3. If a review is pending, read the `next_expected_artifact` path (e.g., `tasks/v0.8-mX-result-review-decision.json`).
  4. Read the corresponding `*-packet.json` and `*-contract.json` to gather review context.
  5. **The Isolation Boundary:** Spawn a completely detached, isolated LLM sub-process (e.g., using `execSync('opencode run ...')` or a native MCP LLM call if configured in `registry.json`) to perform the evaluation.
  6. The isolated process writes the `decision.json` directly to disk.
  7. The script reads the `decision.json`. If `Approved`, it autonomously advances `dispatch.json` to the `execution_ready` or `completed` state. If `Rejected`, it halts and logs the rejection for the primary Executor to fix.

### B. Security & "Agentic Sycophancy" Defenses
- The daemon script itself does *not* evaluate the code. It is merely a dumb orchestrator that spins up a mathematically isolated L3 process.
- The daemon cannot overwrite an explicit `Rejected` decision with an `Approved` state transition. The state machine enforces that the `decision.json` content mathematically dictates the `dispatch.json` advancement.

## 3. Constraints
- **In-Scope:** Writing the `bin/agent-protocol-review` script. Adding the `--install-hook` flag to optionally attach it to Git `pre-commit` for seamless Personal Domain usage.
- **Out-of-Scope:** Implementing Team Mode (Cloud Sync) CI/CD actions (M5). Legacy tooling deprecation (M6). Modifying active workflow state semantics outside of standard M4 progression.
