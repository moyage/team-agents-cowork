# V0.7-M4: Environment Discovery & Capability Modeling

## 1. Objective
Design abstractions to dynamically discover, model, and adapt to heterogeneous AI Coding environments without coupling the core L2/L3 governance to any single toolchain.

## 2. Environment Fingerprint Model
To achieve low invasiveness, the system uses passive "fingerprinting" to deduce the active agent environment. The Coordinator or Evaluator agent reads:
- **Environment Variables:** `OPENCODE_SESSION`, `CURSOR_SESSION_ID`, or proprietary CLI markers.
- **Workspace Anchors:** The presence of `.cursorrules`, `.traerules`, or `CLAUDE.md`.
- **Process Trees:** Detecting active background servers (e.g., `mcp-server` process vs `omoc` daemon).

## 3. Capability Registry
Not all environments can execute all protocol commands. The capability registry maps fingerprints to allowed actions:
- **Opencode:** `supports_mcp: true`, `supports_terminal: true`, `supports_background_process: false`.
- **Cursor:** `supports_mcp: true` (via `.cursorrules`), `supports_terminal: partial` (requires explicit user run).
- **OMOC / Superpowers:** `supports_mcp: false` (uses direct stdio/eval), `supports_background_process: true`.
*(When generating Execution Contracts, the Coordinator selects tasks that fit the Executor's capability profile).*

## 4. Compatibility & Drift Analysis Method
AI toolchains evolve rapidly, breaking adapter integration. The framework monitors for drift:
- **Static Drift:** The framework compares the checksum of the injected `.cursorrules` against the `adapters/CURSOR_RULES.tpl` source.
- **Runtime Drift:** The Evaluator monitors MCP execution logs. If a tool that previously returned JSON suddenly returns a raw string or throws an undocumented error, a Drift Event is logged.

## 5. Adapter Abstraction Rules
- **Rule 1 (Agnostic Core):** The `dispatch.json` state machine and `decision.json` artifacts are universally formatted JSON. No tool-specific schemas are allowed here.
- **Rule 2 (Thin Translators):** Adapters (e.g., `CLAUDE.md`) only serve to translate the environment's system prompt paradigm into an instruction to "Read `documentation/AI_GUIDES/SYSTEM_PROMPT.md` and use MCP tools."
- **Rule 3 (Bounded Adapter Evolution):** When drift is detected, the self-evolution loop generates an improvement contract strictly bounded to `adapters/*.tpl`. It cannot rewrite the core protocol to fix a tool's bug.
