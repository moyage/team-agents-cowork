# V0.7-M2 Reconnaissance: Harness / Workflow / Acceptance Unified Model

## 1. Objective
Translate the abstract roles and scenarios established in M1 into a concrete, executable unified model. This includes defining the artifact taxonomy, the standard task lifecycle, and delineating the semantic boundaries between Review, Gate, Evaluation, and Acceptance.

## 2. Analytical Scope
To ensure the protocol is actionable rather than a mere glossary, we must address the fundamental definitions that govern the L2/L3 state machine without altering the active implementation code (`src/` or `workflow/`).

### Required Deliverables
A comprehensive design document (`tasks/v0.7-m2-unified-model.md`) that answers:
1. **Artifact taxonomy:** What is the absolute minimal set of files?
2. **Task lifecycle:** What are the standard stages?
3. **Reviewer vs. Gatekeeper:** Are they distinct?
4. **Acceptance vs. Evaluation:** Where is the boundary?
5. **Personal vs. Team Mode:** What must be unified vs. locally adapted?
6. **Low Cognitive Load:** How to prevent process bloat?

## 3. Constraints
- **In-Scope:** Generating `tasks/v0.7-m2-unified-model.md`.
- **Out-of-Scope:** Physical `.agent-state/` migration, modifying MCP server runtime logic, implementing automation scripts, modifying the active `dispatch.json` state semantics, or building environment adapters.
