# V0.8 Milestone Map

## V0.8-M1 — The Agent Workspace (.agent-state/ Migration)
- **Objective:** Implement the physical extraction of the execution state (`workflow/` and `tasks/`) to the target repository's `.agent-state/` directory as designed in V0.7-M5-A.
- **In-Scope:** Modifying `src/mcp-server/src/index.ts` to dynamically resolve paths using `process.env.PROJECT_ROOT/.agent-state/` with backward compatibility to `workflow/`. Modifying `bin/agent-protocol-init` to scaffold `.agent-state/`.
- **Out-of-Scope:** Deleting the framework's own `workflow/` or `tasks/` history.
- **Recommended Order:** 1

## V0.8-M2 — The Dynamic Capability Registry
- **Objective:** Implement the environment fingerprinting and capability modeling registry (`.agent-state/registry.json`) designed in V0.7-M4.
- **In-Scope:** Building the initialization logic (via `bin/agent-protocol-init` or the MCP server) to passively detect the active AI Coding environment (e.g., `OPENCODE_SESSION`, `.cursorrules`) and register its capabilities (`supports_mcp`, `is_isolated_reviewer`).
- **Out-of-Scope:** Locking the framework to a specific IDE/CLI.
- **Recommended Order:** 2

## V0.8-M3 — SubAgent Review Automation Daemon
- **Objective:** Implement the background hook or CLI command (`agent-protocol-review`) that autonomously invokes the isolated Gatekeeper upon detecting a `_review_pending` state transition in `.agent-state/workflow/dispatch.json`.
- **In-Scope:** Writing a local Node.js script or Git pre-commit hook that triggers an isolated LLM (e.g., `opencode run -m "openai/gpt-5.4"`) to execute the Gatekeeper review protocol and write the `decision.json` artifact.
- **Out-of-Scope:** Allowing the primary Executor agent to bypass the isolated sub-process and self-approve.
- **Recommended Order:** 3

## V0.8-M4 — Team Mode Orchestration (Dual-Tiered Sync)
- **Objective:** Implement the decentralized "Pull" model for task claiming and state synchronization across remote team members as defined in the V0.8 Planning Q&A.
- **In-Scope:** Defining the standard Git workflow for syncing the `.agent-state/` directory (e.g., a dedicated `agent-sync` branch). Updating the documentation and adapters to reflect the decentralized claiming mechanism.
- **Out-of-Scope:** Building a centralized database or real-time message broker.
- **Recommended Order:** 4

## V0.8-M5 — Legacy Tooling Deprecation
- **Objective:** Execute the deprecation warning phase for the legacy Python tooling (`scripts/*.py`) as mapped in V0.7-M5-B.
- **In-Scope:** Injecting formal deprecation warnings into `scripts/doctor.py`, `scripts/verify_protocol.py`, etc.
- **Out-of-Scope:** Physically deleting the scripts.
- **Recommended Order:** 5
