# V0.7-M5-C Reconnaissance: SubAgent Review Automation Governance

## 1. Objective
Design the governance and execution roadmap for automating the invocation of Gatekeeper SubAgents during `_review_pending` state transitions. This solves the "Review Automation" carry-over without compromising the L2/L3 dual-track security model.

## 2. Analytical Scope
- **Trigger Mechanisms:** How to automatically detect state transitions (e.g., via background Coordinator agents or CI pipelines parsing `dispatch.json`).
- **Gating Boundaries:** Ensuring that the automated script only *invokes* the LLM and does not parse or override the LLM's physical `decision.json` output.
- **Verification Methods:** How does the framework verify that an automated Gatekeeper isn't just rubber-stamping the Executor's work?

## 3. Constraints
- **In-Scope:** Drafting the roadmap design document (`tasks/v0.7-m5-c-review-automation-roadmap.md`).
- **Out-of-Scope:** Writing or deploying the actual automation scripts (e.g., GitHub Actions, local daemon scripts) that bypass or alter the Gatekeeper logic. Modifying active `dispatch.json` state semantics.
