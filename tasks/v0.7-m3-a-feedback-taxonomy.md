# V0.7-M3-A: Failure Taxonomy & Ingestion Model

## 1. Feedback Ingestion Model
To build a governed self-evolution loop, the system must collect diagnostic feedback without breaking the L2/L3 governance isolation. Data is passively sourced from physical artifacts and deterministic exit codes:
- **`contract-review-decision.json` (Rejections):** Natural language comments detailing scope boundary violations.
- **`result-review-decision.json` (Rejections):** Natural language comments detailing git diff mismatch or implementation errors.
- **Tool Execution Logs:** Intercepted stderr and exit codes from MCP tools (e.g., `get_execution_diff` failure) or sub-process calls.
- **Test Framework Outputs:** Standard outputs (e.g., Jest, Pytest) run by an Evaluator.

## 2. Failure Taxonomy
Feedback is classified into distinct categories to drive the correct resolution pipeline:
1. **Execution Boundary Violations:** The agent touched files outside `allowed_files`. (Detected during `result_review_pending`).
2. **Scope Overreach:** The agent drafted a contract that includes actions listed in `out_of_scope`. (Detected during `contract_review_pending`).
3. **Syntax/Compilation/Test Errors:** Code fails to compile or fails pre-defined tests. (Detected by Executor prior to pushing result, or by Evaluator).
4. **Environment/Adapter Drifts:** An expected path, CLI tool, or environment variable fails to resolve (e.g., `ENOENT` on `dispatch.json`).

## 3. Retry vs. Improvement-Case Boundary
An agent working on a task cannot endlessly loop. A strict threshold separates a normal "retry" from a "systemic improvement case."
- **Standard Retry (Autonomous Resolution):** 
  - Occurs within the scope of the *current* active task contract.
  - The agent resets the commit (`git reset HEAD~1`) and tries again.
  - *Threshold:* Up to 3 consecutive rejections on the same phase gate (e.g., 3 rejected result reviews).
- **Improvement Case (Escalation / Self-Evolution):**
  - The threshold of 3 retries is breached, OR the failure is classified as an *Environment/Adapter Drift*.
  - The agent must **halt** execution of the current feature task.
  - It creates a formal **Improvement Proposal** (addressed in M3-B) describing the systemic failure (e.g., "The path resolution in index.ts is fragile"). 
  - The current task is parked, and the Orchestrator evaluates scheduling the new Improvement Epic.

## 4. Retrospective Artifact Model
Upon iteration closure (when `dispatch.json` advances to `iteration_completed`), the Coordinator agent automatically compiles the `retrospective-note.md`:
- It parses all `Rejected` decisions logged in the `tasks/` directory.
- It maps these rejections against the Taxonomy.
- High-frequency errors (e.g., "Scope Overreach") are appended to the `iteration-state.json` -> `carry_overs` array as technical debt to be addressed in the next iteration's protocol guidelines or adapter updates.
