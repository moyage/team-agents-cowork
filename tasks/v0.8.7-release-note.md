# Release v0.8.7

## Overview
Iteration V0.8.7 focuses on addressing accumulated technical debt, enhancing architectural integrity, and significantly overhauling the documentation suite to align with the "Harness Engineering" L0-L6 standard.

## Key Changes
- **Dual-Mode Orchestration**: Introduced `blackbox` vs `orchestrated` routing in the YAML DAG schemas, allowing the framework to safely bypass isolation for local low-risk tasks while strictly enforcing MCP Headless Worktrees for team-based asynchronous tasks.
- **Security Remediation**: Eliminated critical shell injection and path traversal vulnerabilities in the MCP server context API (specifically `verify_execution_compliance` and task routing paths).
- **Architecture Drift Fixed**: Replaced the "Mock" `bin/agent-protocol-review` daemon with real dynamic registry querying and L3 gatekeeper dispatching (e.g., executing `opencode-bridge` or `kimi-code`).
- **Documentation Overhaul**: All documentation rewritten to bilingual parity (EN/ZH) and completely voided of placeholders/stubs, incorporating Mermaid flowcharts for architectural clarity.
- **Repository Hygiene**: Version numbering reconciled to `0.8.7` across all package and manifest files. Dead Python scripts and broken `make` targets were pruned.

## Audit & QA
- **E2E Simulation Passed**: Verified fresh initialization (`bin/agent-protocol-init`) cleanly scaffolds the `.agent-state/` without hardcoded agent bindings.
- **Compiler Checks**: `src/mcp-server` successfully compiles via `tsc` without strict errors.
