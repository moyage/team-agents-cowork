# V0.6 Scope & Milestone Map

The scope of Iteration V0.6 encompasses the structural normalization of the `team-agents-cowork` repository, hardening of the MCP context API pathing, robust integration checking, documentation overhaul, and installability scripting. It explicitly integrates the findings from the V0.5 Post-Closure Normalization Study and the V0.5 Retrospective.

## V0.6-M1 — Repository & Workflow Reconnaissance Baseline (Current)
- **Objective:** Map the entire repository structure, define the scope baseline, read all retrospective and carry_over inputs, and formally start the V0.6 workflow.
- **In-Scope:** Kickoff packets, risk registers, scope mapping, and formal kickoff review.
- **Out-of-Scope:** Deleting files, modifying existing documentation, writing code.
- **Recommended Order:** 1 (Prerequisite for all)

## V0.6-M2 — Read-Only Context API Path/Runtime Hardening
- **Objective:** Address V0.5 Carry-Over #1 (Path Resolution Fragility) across the MCP context API.
- **In-Scope:** Identify all relative path resolutions (e.g., `../../`) in `src/mcp-server/src/index.ts` (specifically `get_iteration_state`, `get_execution_diff`). Propose and implement a robust resolution strategy (e.g., resolving from the git root or absolute configuration).
- **Out-of-Scope:** Adding write-capabilities, structural repo changes.
- **Recommended Order:** 2 (Hardens the core engine before moving it around)

## V0.6-M3 — Integration Smoke & Workflow Consistency Checks
- **Objective:** Bolster integration validation repeatability.
- **In-Scope:** Create a suite or script (building on M4's `test-client.ts`) that asserts workflow state consistency and provides a minimal regression checklist for future agents. Address Reviewer Output Delivery and Approval Gating Automation feasibility.
- **Out-of-Scope:** Auto-approval logic that removes the physical JSON decision artifact requirement.
- **Recommended Order:** 3

## V0.6-M4 — Docs Center / Samples / Demo Foundation
- **Objective:** Provide a clear, categorized onboarding path for both humans and AI agents.
- **In-Scope:** Restructure documentation into `HUMAN_GUIDES`, `AI_GUIDES`, and `SAMPLES`. Write/update `QUICKSTART.md`, `ARCHITECTURE.md`, `GOVERNANCE.md`.
- **Out-of-Scope:** Deleting/moving the old files just yet (that happens in M5). This phase is purely writing/generating the new content structure.
- **Recommended Order:** 4

## V0.6-M5 — Repository Normalization & Consolidation
- **Objective:** Formally execute the cleanup and consolidation of the repository.
- **In-Scope:** Global `.DS_Store` purge. Consolidate `audit/`, `qa/`, `release/`, `retros/` into an `.archive/` directory. Archive V0.4/V0.5 `tasks/` artifacts. Replace old documentation with the M4 content.
- **Out-of-Scope:** "Big Bang Refactoring" of the MCP server or protocol schemas. Must be bounded.
- **Recommended Order:** 5 (Must happen after docs are written so we can safely delete old ones)

## V0.6-M6 — Installability / Bootstrap / Comparative Synthesis
- **Objective:** Create 1-click install/deploy scripts and summarize the framework's advantages vs `labs-teamwork-spec`.
- **In-Scope:** Develop bootstrap scripts (`agent-protocol-init`). Differentiate individual vs team deployment paths. Perform structural comparative analysis with `/Users/mlabs/Programs/labs-teamwork-spec` to summarize best practices and adoption strategies.
- **Out-of-Scope:** Changing the other repository (`labs-teamwork-spec`).
- **Recommended Order:** 6 (Must reflect the final, normalized repository structure from M5)
