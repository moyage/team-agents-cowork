# V0.7 Risk Register

This document tracks known risks, dependencies, and blockers for the V0.7 Iteration.

## Technical Risks
- **`.agent-state/` Migration Friction (High):** Moving from a repository-centric history (`tasks/`, `workflow/`) to a distributed `.agent-state/` model involves fundamental changes to MCP server path resolution and bootstrap logic. If not executed smoothly, it could break existing active workflows or orphan historical data.
- **Environment Drift (Medium):** Building adapters for rapidly evolving tools (Opencode, OMOC, Cursor) means the adapters could break frequently. The self-evolution loop must successfully detect and patch these drifts.
- **Review Automation Loops (High):** Automating the Gatekeeper invocation (`_review_pending` -> Auto-Run SubAgent) might inadvertently create infinite approval loops if the SubAgent consistently rejects a bad contract without human intervention or a circuit breaker.

## Governance Risks
- **Autonomy vs. Control (High):** The "Self-Evolution Loop" introduces the risk of "Agentic Sycophancy" if the system learns to automatically write contracts that the Gatekeeper rubber-stamps. The isolation between the Executor and the Gatekeeper must remain mathematically secure.
- **Scope Creep (Medium):** The ambitious goals of V0.7 (multi-agent, multi-environment, self-evolution) could easily expand beyond bounded milestones. Strict adherence to the Milestone-Level Autonomy and Phase Gating protocol is critical.

## Dependency Gaps & Blockers
- **`labs-teamwork-spec` Local Path:** The comparative synthesis and potential integration of scaffolding patterns depend on `/Users/mlabs/Programs/labs-teamwork-spec`. If this path changes or the repo is unavailable, M6 and related design tasks will be blocked.
- **MCP Server Capabilities:** The current MCP server (`src/mcp-server/`) is read-only. V0.7's orchestration and review automation goals might require limited, governed write capabilities (e.g., updating `dispatch.json` state programmatically) which must be carefully designed to avoid bypassing the physical JSON artifact requirements.
