# V0.4 Kickoff & Planning

**启动时间:** YYYY-MM-DD (Pending)
**状态:** 规划中 (Planning)
**核心主题:** From Static Protocol to Automated Orchestration

## 1. 背景与启动依据
随着 V0.3 的正式收口，Team Agent Collaboration Protocol 已经拥有了扎实的“静态规则底盘”：包括远端注入、嗅探、反馈流转和安全防篡改（CI）。但目前的协作依然依赖人类去手动派发每个环节的文档任务。
V0.4 的核心诉求是从“规范提供者”向“流程驱动器”过渡。

## 2. V0.4 候选方向与优先级
基于 `tasks/v0.4-demand-pool.md`，初步圈定以下方向为 V0.4 核心探讨域：
1. **Workflow Orchestrator Evaluation (P0)**: 评估选用何种引擎驱动多 Agent 流转。
2. **多仓库长效同步 (P1)**: 使得分散在各个微服务中的协议底座能够感知主仓库的更新。
3. **Context 压缩与结构化 (P2)**: 保证需求池与迭代报告不至于随着生命周期拉长而撑爆 LLM 的上下文。
