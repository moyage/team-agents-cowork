# V0.7-M5-A: .agent-state/ Separation Roadmap

## 1. Objective
To physically decouple the framework's core engine (MCP server, scripts, documentation) from the user's execution history (`tasks/` and `workflow/` directories). By separating state from the framework, the protocol can be cleanly distributed as an NPM CLI tool and injected into any project without carrying baggage.

## 2. Target State
- **Framework Repo:** Acts solely as the engine source code. The root contains `bin/`, `src/mcp-server/`, `adapters/`, and `documentation/`.
- **Target Project (User Repo):** 
  - An `.agent-state/` folder is generated at the root.
  - `.agent-state/workflow/` contains `dispatch.json` and `iteration-state.json`.
  - `.agent-state/tasks/` contains execution contracts, reviews, and result artifacts.
- **MCP Server:** Will dynamically resolve paths by looking for `.agent-state/workflow/dispatch.json` relative to `process.env.PROJECT_ROOT`.

## 3. Intermediate & Compatibility State
To prevent sudden breakage of existing pipelines or testing environments during the transition:
- The `agent-protocol-init` bootstrap script will generate the `.agent-state/` folder locally inside the framework repo for self-testing.
- However, `.gitignore` will explicitly ignore `.agent-state/` to prevent tracking runtime state in the engine repo.
- **MCP Server Fallback:** The server logic will be modified to check for the existence of `PROJECT_ROOT/.agent-state/workflow/dispatch.json`. If it does not exist, it will silently fall back to `PROJECT_ROOT/workflow/dispatch.json`. This guarantees 100% backward compatibility for environments that have not yet migrated.

## 4. Rollback Path
If the `.agent-state/` model introduces unacceptable friction or breaks IDE integration:
1. **Revert Path Resolution:** Simply remove the primary check for `.agent-state/` in `src/mcp-server/src/index.ts`, returning it strictly to `workflow/`.
2. **Revert Bootstrap:** Remove `.agent-state/` generation from `bin/agent-protocol-init`.
3. **Data Recovery:** For users who migrated, simply `git mv .agent-state/workflow workflow` and `git mv .agent-state/tasks tasks`. Because the schemas remain identical, no JSON data manipulation is required.
