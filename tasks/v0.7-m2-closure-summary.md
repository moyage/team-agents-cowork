## 2. Core Design Summary
The `tasks/v0.7-m2-unified-model.md` artifact formally established the following executable paradigms:
- **Artifact Taxonomy:** Formalized the minimal set of 5 JSON artifacts that drive the state machine: `dispatch.json` (State), `execution-contract.json` (Intent), `contract-review-decision.json` (Pre-Gate), `execution-result.json` (Evidence), and `result-review-decision.json` (Post-Gate).
- **Task Lifecycle:** Mapped the standard progression: `kickoff_pending` -> `recon_completed` -> `contract_review_pending` -> `execution_ready` -> `result_review_pending` -> `completed`.
- **Reviewer vs. Gatekeeper:** Defined "Reviewer" as the evaluative action, while "Gatekeeper" is the protocol function that generates the physical `decision.json` artifact unlocking the state machine block.
- **Acceptance vs. Evaluation:** Clarified that "Evaluation" is the technical assessment of implementation (compilation, boundary compliance), whereas "Acceptance" is the human/business-level sign-off against the Epic's product intent.
- **Handoff Rules (Personal vs. Team Mode):** Mandated that the 5 JSON artifacts and the `dispatch.json` state machine are the **Unified Common Core**. In Personal Mode, Gatekeeping is automated via local hooks for low-friction velocity; in Team Mode, Gatekeeping shifts to CI pipelines and dedicated Coordinator Agents for robust synchronization.
