# V0.6-M3 Reconnaissance: Documentation Center Unification

## 1. Objective
Establish a unified Documentation Center (`documentation/`) featuring dual-track consumption paths for Human Collaborators (`HUMAN_GUIDES/`) and AI Integration Environments (`AI_GUIDES/`). Refactor the root `README.md` to serve solely as a minimalist entry point. Establish a `SAMPLES/` directory to index real-world execution artifacts without mistaking them for active workflow state.

## 2. Current State Analysis
- **Root `README.md`:** Currently extremely verbose (bloated with onboarding, architecture explanations, and script usage). It attempts to serve every audience simultaneously, resulting in a high cognitive load.
- **Existing `docs/`:** Contains `ONBOARDING.md`, `QUICKSTART.md`, `TROUBLESHOOTING.md`, etc.
- **Existing `protocol/`:** Contains the L0-L6 architecture rules (`01-collaboration-protocol.md` through `08-multi-iteration.md`).
- **Samples/Demos:** V0.5 iteration artifacts currently bloat the active `tasks/` directory. They need a formalized destination and index structure within `documentation/SAMPLES/` to act as Few-Shot learning material for future agents.

## 3. Structural Blueprint
Based on the Post-V0.5 Study, the target `documentation/` structure is:
```
documentation/
├── HUMAN_GUIDES/
│   ├── QUICKSTART.md
│   ├── ARCHITECTURE.md
│   ├── GOVERNANCE.md
│   └── FAQ_TROUBLESHOOTING.md
├── AI_GUIDES/
│   ├── SYSTEM_PROMPT.md
│   ├── MCP_API_REFERENCE.md
│   └── SCHEMA_REFERENCE.md
└── SAMPLES/
    └── INDEX.md
```

## 4. Scope & Governance Boundaries
- **In-Scope:** Creating the `documentation/` directory hierarchy. Populating the new markdown files by extracting and restructuring content from `README.md`, `docs/`, and `protocol/`. Trimming `README.md` to a minimalist entry point. Establishing migration mappings for old documents.
- **Out-of-Scope (Strict Constraints):**
  - **No Deletion/Movement of Existing Files:** To prevent breaking existing hyperlinks prematurely, the old `docs/` and `protocol/` files will remain exactly where they are. Their deletion is deferred to V0.6-M6 (Repository Normalization cleanup).
  - **No Bootstrap Scripts:** The `QUICKSTART.md` will describe the conceptual setup. Actual bootstrap scripts (`agent-protocol-init`) belong to M4.
  - **No MCP/Runtime Changes:** This is purely a documentation generation milestone.
  - **No Execution Artifact Archival:** The `tasks/` directory will not be touched. `documentation/SAMPLES/INDEX.md` will simply point to the current location of the V0.5 artifacts or explain where they will eventually be archived.
