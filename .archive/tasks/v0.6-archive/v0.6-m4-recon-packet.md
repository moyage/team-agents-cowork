# V0.6-M4 Reconnaissance: Bootstrap & Enablement

## 1. Objective
Design and implement 1-click installation and configuration scripts (`agent-protocol-init`) for individual and team modes. This script must successfully initialize the framework state using the newly unified documentation paths.

## 2. Current State Analysis
- **Existing Scripts:**
  - `install.sh` / `install.ps1`: Legacy installers that clone the repo and sniff for IDEs (`.cursorrules`, etc.).
  - `setup.sh`: A local initialization script.
- **Deficiencies:**
  - Existing scripts do not initialize or manage the MCP server (`npm ci`, `npm run build`).
  - They point to deprecated documentation paths (`docs/`, `protocol/`) rather than the new `documentation/HUMAN_GUIDES` and `documentation/AI_GUIDES`.
  - They do not properly differentiate between a "Local Dev/Demo" mode (which initializes a blank `workflow/dispatch.json`) and a "Team Integration" mode (which points agents to a shared truth).

## 3. Structural Blueprint
The target `bin/agent-protocol-init` script (or `install.sh` rewrite) must perform:
1. **Environment Check:** Validate Node.js (for MCP) and Python (if legacy tooling remains).
2. **MCP Initialization:** `npm ci` and `npm run build` inside `src/mcp-server/`.
3. **Workspace Injection:** Automatically inject `.cursorrules`, `CLAUDE.md`, or `AGENTS.md` into the user's workspace, redirecting prompts to `documentation/AI_GUIDES/SYSTEM_PROMPT.md`.
4. **State Generation:** 
   - *Individual Mode:* Wipe the `tasks/` directory of framework history and generate a clean `workflow/dispatch.json` (e.g., `iteration: V1.0`, `gate: kickoff_pending`).
   - *Team Mode:* Validate existing state paths without wiping history.

## 4. Scope & Governance Boundaries
- **In-Scope:** Creating/modifying shell scripts (`install.sh`, `setup.sh`, `bin/agent-protocol-init`). Updating adapter templates (`adapters/*.tpl`) to point to the new Docs Center paths.
- **Out-of-Scope:** Modifying MCP Server logic (deferred to M5). Removing legacy Python scripts (deferred to M5). Refactoring the actual Markdown text of the docs.
