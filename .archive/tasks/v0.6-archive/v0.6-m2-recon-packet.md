# V0.6-M2 Reconnaissance: Historical Archive Consolidation

## 1. Objective
Consolidate historical point-in-time documents (`audit/`, `qa/`, `release/`, `retros/`) into a centralized `.archive/` directory to reduce root-level repository noise, while preserving full audit traceability and read history.

## 2. Current State Analysis
- **Directories Identified:**
  - `audit/`: 5 markdown reports (consistency, risk, findings).
  - `qa/`: 4 markdown reports (acceptance, validation).
  - `release/`: 8 markdown files (release notes, checklists).
  - `retros/`: Iteration archives (v0.2, v0.3, v0.4) containing extensive historical planning and implementation records.
- **Cross-References:** `git grep` reveals minor cross-referencing within historical files (e.g., `protocol/08-multi-iteration.md` refers to `retros/`, and older retrospectives refer to themselves). 
- **Risk Assessment:** Modifying the content of historical files to fix paths is an anti-pattern (it alters audit evidence). Instead, moving the folders cleanly via `git mv` and providing an `.archive/README.md` (Manifest/Index) ensures that anyone reading an old reference knows the target directory has simply been nested.

## 3. Scope Definition
- **In-Scope:**
  - `git mv audit/ .archive/audit/`
  - `git mv qa/ .archive/qa/`
  - `git mv release/ .archive/release/`
  - `git mv retros/ .archive/retros/`
  - Create `.archive/README.md` containing a manifest and mapping of the consolidated artifacts.
- **Out-of-Scope:**
  - Deleting evidence.
  - Rewriting historical markdown content.
  - Moving `tasks/`, `workflow/`, or `src/` artifacts.
  - Restructuring the Documentation Center (M3/M4 scope).
