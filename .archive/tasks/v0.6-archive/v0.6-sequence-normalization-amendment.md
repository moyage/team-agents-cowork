# V0.6 Sequence Normalization Amendment (Post-M3)

## 1. Rationale for Normalization
The initially approved V0.6 Kickoff defined the milestone map with an API-hardening-first sequence (M2: Path Hardening, M3: Integration Smoke, M4: Docs, M5: Repo Normalization, M6: Bootstrap). However, under the authorized Autonomous Milestone Mode, the execution naturally adopted a repository-first cleanup and foundation-laying sequence to reduce cognitive load before touching code or integration scripts. 

To ensure the canonical milestone map explicitly reflects the executed reality without dropping any approved technical objectives or V0.5 carry-overs, this amendment formally normalizes the V0.6 sequence.

## 2. Executed Facts (Normalized Sequence)
The following milestones have been executed and closed under the new sequence:
- **V0.6-M1 (Completed): Repository Hygiene**
  - *Fact:* Globally purged `.DS_Store` files and updated `.gitignore`.
- **V0.6-M2 (Completed): Historical Archive Consolidation**
  - *Fact:* Relocated `audit/`, `qa/`, `release/`, and `retros/` to `.archive/` with a manifest.
- **V0.6-M3 (Completed): Documentation Center Unification**
  - *Fact:* Rebuilt the docs center into `HUMAN_GUIDES`, `AI_GUIDES`, and `SAMPLES`, and refactored `README.md` as a minimalist navigator.

## 3. Revised Canonical Milestone Map (M4-M6)
The original kickoff commitments (Path/Runtime Hardening, Integration Smoke/Workflow Consistency) are NOT deleted. They are explicitly incorporated into the remaining milestones as mandatory workstreams and exit criteria.

- **V0.6-M4 (Next): Bootstrap & Enablement**
  - *Objective:* Design 1-click installation and configuration scripts (`agent-protocol-init`) for individual and team modes.
  - *Mandatory Exit Criteria:* The bootstrap process must successfully initialize the framework state using the newly unified documentation paths.
- **V0.6-M5: Tooling Deprecation & Read-Only API Hardening**
  - *Objective:* Address V0.5 carry-overs related to path resolution fragility (`get_iteration_state`, `get_execution_diff`) across the MCP context API. Evaluate legacy Python script deprecation in light of the new docs center.
  - *Mandatory Workstream:* Must implement robust runtime hardening for the MCP server paths to support the bootstrap configurations established in M4.
- **V0.6-M6: Integration Smoke, Workflow Consistency & Comparative Synthesis**
  - *Objective:* Build an integration smoke test suite asserting workflow state consistency. Address remaining V0.5 carry-overs (Reviewer Output Delivery, Approval Gating Automation). Finalize framework vs. execution-state separation design and summarize differences against `labs-teamwork-spec`. Perform the final artifact archival (moving V0.4/V0.5 `tasks/` to `.archive/`) and purge redundant old docs.

## 4. Governance Note
This normalization guarantees that the physical execution history (M1-M3) perfectly aligns with the canonical project plan, while strictly preserving all V0.6 kickoff commitments and V0.5 carry-overs within the upcoming M4-M6 boundaries.
