# V0.5 M5 Kickoff & Reconnaissance Note

## 1. Objective
Fix the technical debt introduced in M3 and exposed during M4 integration testing: the path resolution error (`ENOENT`) in the `get_dispatch_state` MCP tool.

## 2. Root Cause Analysis (Bounded Reconnaissance)
- **Observation:** M4's `test-client.ts` received the error: `ENOENT: no such file or directory, open '/Users/mlabs/Programs/team-agents-cowork/src/workflow/dispatch.json'`.
- **Investigation:** In `src/mcp-server/src/index.ts`, `__dirname` resolves to `[ProjectRoot]/src/mcp-server/src`. 
- **The Bug:** The code attempts to resolve the dispatch file using `path.resolve(__dirname, "../../workflow/dispatch.json")`. Moving up two directories (`../../`) lands in `[ProjectRoot]/src`, not `[ProjectRoot]`. Consequently, it looks for `src/workflow/dispatch.json` which does not exist.
- **The Fix:** The path resolution needs to traverse up three directories: `path.resolve(__dirname, "../../../workflow/dispatch.json")`.

## 3. Scope & Boundary Assessment
- **`allowed_files` Expansion Check:** The existing `test-client.ts` was written robustly to print whatever the server returns (error or success). Therefore, **no modifications to `test-client.ts` are required**. The `allowed_files` boundary can remain strictly limited to `src/mcp-server/src/index.ts`.
- **Non-blocking Issues Exclusion:** The bug also technically applies to `projectRoot` used for `git diff` and `WORKFLOW_STATE_PATH` used for `get_iteration_state`. However, since `git diff` works anywhere inside a git repository, the `git diff` command did not fail in M4. Per the strict rule "不得顺手修复其他非阻塞问题" (Do not fix non-blocking issues incidentally), we will **only** modify the exact line causing the `get_dispatch_state` failure.
- **Read-Only Semantic:** The fix only changes a string path passed to `fs.readFile`. It introduces zero write capabilities.

## 4. Execution & Closure Rules
- Implementation must only touch the single path resolution line.
- The closure requires generating a standard execution result, running the test client to prove the fix, and obtaining an Independent Reviewer's Approval before updating workflow states.
