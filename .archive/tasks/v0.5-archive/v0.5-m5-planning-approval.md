# M5 Planning Approval

## Decision
**Status:** Approved

## Rationale
The M5 objective is explicitly defined as a bounded restoration bugfix to correct a path resolution error (`ENOENT`) within the `get_dispatch_state` MCP tool. It does not introduce new features, expand the toolset, or alter existing L2/L3 governance models. By isolating the fix strictly to the broken `dispatchPath` string and expressly forbidding the incidental repair of non-blocking paths (e.g., `projectRoot` or `WORKFLOW_STATE_PATH`), the milestone remains perfectly constrained and compliant with the "no arbitrary technical debt cleanup" directive.

## Governance Directives
1. **Strictly Read-Only Restoration:** The implementation merely corrects a relative directory jump (`../../` to `../../../`) passed to `fs.readFile`. It introduces zero write capabilities.
2. **Hard Isolation:** `allowed_files` must only contain `src/mcp-server/src/index.ts`. The existing test client (`test-client.ts`) is strictly out of bounds for modification and should only be executed to verify the fix.
3. **No Incidental Fixes:** The contract must explicitly forbid patching other tools' paths or schemas.
