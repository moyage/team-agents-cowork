# M4 Planning Approval

## Decision
**Status:** Approved

## Rationale
The proposed M4 objective—a read-only, programmatic integration test client (`src/mcp-server/src/test-client.ts`)—is a perfectly bounded, integration-focused milestone. It "dogfoods" the existing MCP server's M3 and M2 tools to validate that an orchestrator/reviewer agent can successfully extract the canonical workflow state (`get_dispatch_state`) and semantic diffs (`get_execution_diff`). This proves the L2/L3 integration viability over the stdio transport layer without expanding the MCP protocol or mutating workflow states.

## Governance Directives
1. **Strictly Read-Only Target:** The client script must only consume data via the MCP tools and print/assert the results. It must not contain logic to write back to `dispatch.json` or generate `decision.json` artifacts automatically.
2. **No Protocol Overhaul:** The existing MCP server (`src/mcp-server/src/index.ts`) must not be modified. If the existing M2/M3 tools are found insufficient, the M4 milestone must halt and log an exception rather than extending the protocol.
3. **Artifact / File Separation Enforced:** `allowed_files` is strictly limited to the new client script. `allowed_artifacts` captures the M4 execution result and review packet.
