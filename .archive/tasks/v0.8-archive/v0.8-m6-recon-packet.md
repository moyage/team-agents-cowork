# V0.8-M6 Reconnaissance: Legacy Tooling Deprecation

## 1. Objective
Execute the formal deprecation of the legacy Python tooling (`scripts/*.py`) as mandated by the V0.7-M5-B roadmap. The objective is to inject deprecation warnings into these scripts to warn downstream users or CI pipelines that these tools will be completely removed in Iteration V0.9, as their functionality is now fully superseded by the Node.js MCP server.

## 2. Analytical Scope

### Target Files
- `scripts/doctor.py`
- `scripts/verify_protocol.py`
- `scripts/validate_workflow_state.py`

### Mechanism
- Inject a clear, multi-line `print` or `sys.stderr.write` warning at the very beginning of the `if __name__ == '__main__':` block of each targeted script.
- The warning must state:
  - The script is deprecated.
  - The functionality has moved to the `src/mcp-server/` logic.
  - The script will be permanently deleted in Iteration V0.9.
- Allow the script to continue execution normally after printing the warning (to prevent breaking any external pipelines that still rely on it during the V0.8 transition).

## 3. Constraints
- **In-Scope:** Modifying the Python scripts to include the print warning.
- **Out-of-Scope:** Physically deleting the Python files. Modifying the MCP server logic. Modifying active workflow state semantics outside of M6 progression.
