# V0.8 Risk Register

This document tracks known risks, dependencies, and blockers for the V0.8 Iteration.

## Technical Risks
- **`.agent-state/` Migration Friction (High):** Moving from a repository-centric history (`tasks/`, `workflow/`) to a distributed `.agent-state/` model involves fundamental changes to MCP server path resolution and bootstrap logic. If not executed smoothly, it could break existing active workflows or orphan historical data.
- **Environment Drift (Medium):** Building adapters for rapidly evolving tools (Opencode, OMOC, Cursor) means the adapters could break frequently. The capability registry and adapter templates must be robustly tested.
- **Review Automation Loops (High):** Automating the Gatekeeper invocation (`_review_pending` -> Auto-Run SubAgent) might inadvertently create infinite approval loops if the SubAgent consistently rejects a bad contract without human intervention or a circuit breaker.
- **Git History Pollution (Medium):** In Team Mode, syncing the `.agent-state/` directory across developers could lead to massive merge conflicts if not managed via a dedicated branch (e.g., `agent-sync`) or if intermediate artifacts (like rejected contracts) are aggressively pushed.

## Governance Risks
- **Autonomy vs. Control (High):** The "SubAgent Review Automation" introduces the risk of "Agentic Sycophancy" if the system learns to automatically write contracts that the background Gatekeeper rubber-stamps. The isolation between the Executor and the Gatekeeper must remain mathematically secure.
- **Scope Creep (Medium):** The ambitious goals of V0.8 (dynamic orchestration, `.agent-state/` migration, automated reviews) could easily expand beyond bounded milestones. Strict adherence to the Milestone-Level Autonomy and Phase Gating protocol is critical.

## Dependency Gaps & Blockers
- **`labs-teamwork-spec` Local Path:** The comparative synthesis and potential integration of scaffolding patterns depend on `/Users/mlabs/Programs/labs-teamwork-spec`. If this path changes or the repo is unavailable, orchestration logic extraction will be blocked.
- **OpenCode Background Execution:** The automated SubAgent review relies heavily on `opencode run -m "openai/gpt-5.4" ...` being available in the environment path and executing flawlessly as a detached container. If this command fails or loses context, the iteration will stall in `review_pending`.
