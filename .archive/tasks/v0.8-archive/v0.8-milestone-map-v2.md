# V0.8 Amended Milestone Map: The Heterogeneous DAG Engine

This document replaces the original `v0.8-milestone-map.md`. It formally integrates the massive architectural shift toward a YAML-based Directed Acyclic Graph (DAG) workflow engine, an LLM Router, and the porting of Archon's 17 default workflows.

## V0.8-M1 — The Agent Workspace (.agent-state/ Migration & Core DAG Parser)
- **Objective:** Implement the physical extraction of the execution state (`workflow/` and `tasks/`) to `.agent-state/`. Build the foundational parser capable of reading a YAML DAG definition (the "Archon-lite" engine) and mapping it to the `team-agents-cowork` JSON contract taxonomy.
- **In-Scope:** Path resolution updates in `src/mcp-server/src/index.ts`, scaffolding `.agent-state/workflows/`, and defining the initial YAML syntax.
- **Recommended Order:** 1

## V0.8-M2 — The LLM Router & Dynamic Capability Registry
- **Objective:** Implement the dynamic workflow router and environment registry (`.agent-state/registry.json`).
- **In-Scope:** Building the initialization logic (detecting `OPENCODE_SESSION`, `.cursorrules`), and creating the LLM prompt that evaluates a user's intent against all available YAML workflows to output a single execution path.
- **Recommended Order:** 2

## V0.8-M3 — Built-in Workflow Porting (The "Archon 17")
- **Objective:** Port Archon's ~17 default workflows (e.g., `archon-fix-github-issue`, `archon-plan-to-pr`, `archon-architect`) into our federated JSON-contract model.
- **In-Scope:** Writing the 17 default `.yaml` workflow files that utilize `team-agents-cowork` L2/L3 dual-track gating and capability-based routing (rather than centralized LLM calls).
- **Recommended Order:** 3

## V0.8-M4 — SubAgent Review Automation Daemon
- **Objective:** Implement the background hook or CLI command (`agent-protocol-review`) that autonomously invokes the isolated Gatekeeper upon detecting a `_review_pending` state transition generated by a YAML node.
- **In-Scope:** Writing a local Node.js script or Git pre-commit hook that triggers an isolated LLM (e.g., `opencode run`) to execute the Gatekeeper review protocol.
- **Recommended Order:** 4

## V0.8-M5 — Custom Workflow Authoring & Team Mode Sync
- **Objective:** Validate the custom workflow authoring capability for users and implement the decentralized "Pull" model (Dual-Tiered Sync) across remote team members.
- **In-Scope:** Defining the Git workflow for syncing `.agent-state/` (`agent-sync` branch), updating adapters, and writing custom workflow authoring documentation.
- **Recommended Order:** 5

## V0.8-M6 — Legacy Tooling Deprecation
- **Objective:** Execute the deprecation warning phase for the legacy Python tooling (`scripts/*.py`).
- **In-Scope:** Injecting formal deprecation warnings into scripts.
- **Recommended Order:** 6
