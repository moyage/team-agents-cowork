# V0.3 Backlog

## 1. 特别声明：V0.2 -> V0.3 作为“多轮迭代演进”的真实实践样本
本次 V0.3 的规划过程本身，是 **“多轮迭代项目演进支持”** 能力的第一批实践落地。我们在此次转接中，需要重点观察并记录以下实践点：
- **发布后如何进入下一轮规划**：从 Release Tag 锁定，到回撤重整 backlog 的全过程。
- **如何从 V0.2 文档继承到 V0.3 backlog**：避免重新发明轮子，利用既有的 `tasks/` 目录进行上下文切换。
- **如何处理上一轮 deferred items**：不再是直接遗忘，而是通过 `deferred-to-v0.x.md` 的机制进行边界界定与重新提权。
- **如何避免第二轮迭代时目录/文档混乱**：通过明确的 `retros/` 存放上期总结，`tasks/` 存放下期池子，用归档与前缀区分。
- **如何记录迭代之间的关系和边界**：要求有明确的修订记录与收口报告。

我们将在 V0.3 迭代结束时，将以上实践抽象为标准的 `multi-iteration-guide.md` 协议，以支持所有采用本协议的团队。

## 2. 🔴 Committed for V0.3 (必须完成范围)

### 1. 中央化远端分发 (Remote Distribution)
- **问题描述**：当前用户必须 `git clone` 整个仓库再执行脚本，流程过重。
- **用户/团队价值**：将 3 分钟的接入时间压缩到 30 秒内，实现 `curl | bash` 的极简下发。
- **建议验收口径**：在一台干净的新机器上，只需一行 `curl` 命令即可将协议规范注入目标空目录，且结构完整。
- **优先级**：P0
- **依赖关系**：无，作为 V0.3 第一切入点。

### 2. 自动环境嗅探 (Auto-Sniffing)
- **问题描述**：手动传参 `--ide cursor` 容易输错，且与远端静默下发模式冲突。
- **用户/团队价值**：脚本自动识别目录特征（如 `.cursor/`），无需人类干预。
- **建议验收口径**：在存在特定 IDE 配置的目录下执行远端下发时，如果不带参数，能自动精准部署对应的 legacy/profile。
- **优先级**：P0
- **依赖关系**：依赖“远端分发”脚本的主体重构。

### 3. 多轮迭代项目演进支持 (Multi-Iteration Evolution)
- **问题描述**：协议此前偏向“一锤子买卖”，缺乏 v1 到 v2 演进的上下文继承机制。
- **用户/团队价值**：团队在长时间跨度维护一个项目时，不至于丢失知识库脉络。
- **建议验收口径**：在协议目录中输出专门的《多轮迭代协同范式（模板与约定）》，并在 V0.3 开发本身通过其检验。
- **优先级**：P1

### 4. 流程中途上游返修支持 (Upstream Rework Support)
- **问题描述**：Build 阶段发现之前的 Spec 不行，不知道该怎么合规打回。
- **用户/团队价值**：容错性极大地增强，适配现实开发中复杂的返工场景。
- **建议验收口径**：补充 `rework-protocol.md` 或在流程节点上允许“后一阶段”打回给“前一阶段”，并定义打回时的交付物质变。
- **优先级**：P1

### 5. 反馈收集与需求池机制 (Feedback & Backlog Mechanism)
- **问题描述**：缺乏收集试点人员痛点的标准途径。
- **用户/团队价值**：使项目组可以持续修正团队规范，形成闭环。
- **建议验收口径**：产出规范化的反馈 Issue 模板或反馈目录，并在工具链中添加快捷提取入口。
- **优先级**：P1

### 6. 防篡改 CI (Anti-Tampering CI)
- **问题描述**：团队成员在本地强行删除注入的协议约束，进而污染代码库。
- **用户/团队价值**：刚性治理，保障架构底线。
- **建议验收口径**：包含一套可复用的 GitHub Action 模板，能对团队的核心 `.cursorrules` 等文件进行 Hash 或变更拦截。
- **优先级**：P2
- **依赖关系**：应放在所有无摩擦体验能力（分发、嗅探）建设完毕之后进行。

## 4. 🔵 Later / Post-V0.3 (明确不进入本期迭代)

### 1. 多 repo 同步中心
- **问题描述**：统一管理企业内部数十个微服务的规则更新。
- **为什么不进当前 V0.3**：投入产出比过低，偏向重度企业级 SaaS 能力，违背当前单仓轻量试点的推广节奏。

### 2. 更深诊断能力
- **问题描述**：深插到具体 IDE (如 Cursor) 里的使用追踪或质量评价钩子。
- **为什么不进当前 V0.3**：强依赖 IDE 闭源环境，极大概率遭遇私有 API 阻断，且引致隐私合规问题。优先用标准 CI/CD 解决质量关。
