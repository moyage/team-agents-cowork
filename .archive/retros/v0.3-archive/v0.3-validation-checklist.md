# V0.3 Validation Checklist (验证点核对清单)

## [Phase 1 / P0: Onboarding UX]
- [x] **网络分发入口 (install.sh)**
  - 方法: 代码审查与沙箱演练，验证是否使用 `mktemp` 创建无污染的隔离层。
  - 证据: `install.sh` 头部的条件判定与目录生成逻辑。
- [x] **自动环境嗅探 (setup.sh --ide auto)**
  - 方法: 创建 `test-auto-sniff/.cursor` 伪目标后运行 `./setup.sh --target ... --ide auto`。
  - 证据: 终端正确输出 `Auto-sniffed IDE environment: cursor`。

## [Phase 2 / P1: Collaboration Loop]
- [x] **摩擦与问题模板存在性**
  - 方法: 文件探测。
  - 证据: `templates/PROBLEM_REPORT.md`, `templates/FRICTION_LOG.md` 存在且格式标准。
- [x] **需求池聚合脚本可用性**
  - 方法: 运行 `python3 scripts/build_demand_pool.py`。
  - 证据: 成功在 `tasks/demand-pool.md` 产生聚合。
- [x] **返修与上游机制流转逻辑**
  - 方法: 检查 `protocol/07-feedback-and-rework.md` 及对应模板。
  - 证据: 明确标识 `draft` -> `in-review` -> `accepted`。
- [x] **多轮迭代状态更新能力**
  - 方法: 检查 `workflow/iteration-state.json` 的表达正确性。
  - 证据: 解析出的 JSON 内容准确指明前置迭代 V0.2 与本轮迭代 V0.3。

## [Phase 3 / P2: Anti-Tamper CI]
- [x] **防篡改基线建立**
  - 方法: 检查 `ci/integrity-manifest.json` 与 `docs/anti-tamper-ci.md`。
  - 证据: 10 个受保护文件定义清晰并包含其 SHA-256。
- [x] **本地篡改检测防御**
  - 方法: 修改 `setup.sh` 文本，运行 `verify_integrity.py` 脚本，随后再复原重测。
  - 证据: 测试期间脚本立刻报出 `INTEGRITY CHECK FAILED` 并在复原后显示 `passed`。
- [x] **GitHub Actions 流水线就绪**
  - 方法: 检查 `.github/workflows/protocol-integrity.yml`。
  - 证据: 含有利用 Python 检测安全指纹并上传 `integrity-result.json` Artifact 的流水线代码。
